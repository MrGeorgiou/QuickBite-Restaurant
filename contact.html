Okay, okay, I have a confession to make...

QuickBite isn't real!

As you may or may not be aware, QuickBite isn't real. QuickBite is a concept for a website, which was created as a web-development project, to practice and showcase a static website using HTML, CSS and JavaScript.

  • The Basics

    The goal of this project was to use the basics of front-end development and to use them well.

    There is alot of HTML in this project and i have tried to format it as well as is possible. The indentation in the code is neat and tidy, the HTML itself is properly applied and the code is readable. A broad variety of html elements and techniques were used in this project, the menu page uses multiple heading types and nested unordered lists to create a complex yet easily readable menu. Grids and tables were also used.

    The CSS in this project is split across multiple files making it easy to maintain and make changes.

    The JavaScript is simple yet effective, it is used to create the parallax effect on the About-us page and to display the navigation menu on smaller screens.

  • Design

    I didn't want to build a generic restaurant, I wanted to create something that was a little less serious and explore designing a logo aswell. I downloaded Adobe Illustrator and designed multiple svgs to use as logos and learned alot in the process. The original svg I found online and purchased then i modified it in Illustrator untill I was happy to use the logo.

    I had so much fun designing the logo, that I started looking for another challenge. I had some familiarity with parallax effects before this project but had never built one myself. I found an image I had taken back in 2020 of the local lake and decided it was exactly what I was looking for. I isolated the different aspects of the image by drawing paths in Adobe Illustrator, then I used an online tutorial to teach me what I needed to create the effect. Using JavaScript to identify the scroll position on the page and updating the position of each aspect of the image relative to that scroll postion. Below I have attached the original image.

  • Responsivity

    The final goal of this project was to create a web-site that renders perfectly and looks the same on any screen-size with as little additional code as possible. To achieve this I explored using different measurement sizes in css combined with svgs for the logos and a couple of media queries to ensure everything looked right. I'm really happy with the end result, I've learned alot and now have the challenge of refactoring my other projects to bring them up to this level.
